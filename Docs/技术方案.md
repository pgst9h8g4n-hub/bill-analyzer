# 消费账单分类分析系统 - 技术方案

## 一、项目概述

多平台消费账单（支付宝/微信/抖音）的统一解析、自动分类、统计分析工具。支持多用户（家庭成员），自定义分类体系，Web 端展示。

#### 2.2.1 `users` — 用户表

```sql
CREATE TABLE users (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name        TEXT NOT NULL,              -- 显示名（如"刘雅思"、"bref."）
    phone       TEXT UNIQUE,                -- 支付宝账号/手机号（用于匹配账单归属）
    email       TEXT UNIQUE,                -- Supabase auth 关联
    avatar_url  TEXT,
    created_at  TIMESTAMPTZ DEFAULT now(),
    updated_at  TIMESTAMPTZ DEFAULT now()
);

-- RLS: 用户只能看到自己的数据
ALTER TABLE users ENABLE ROW LEVEL SECURITY;
CREATE POLICY "users_select_own" ON users
    FOR SELECT USING (auth.uid()::text = id::text OR auth.uid() IS NULL);
```

> **说明**：phone 字段用于自动识别账单归属。支付宝导出文件包含手机号，微信包含昵称，抖音包含证件号。解析时提取这些标识自动关联用户。

### 2.3 查询性能优化

```sql
-- 按月统计的物化视图
CREATE MATERIALIZED VIEW IF NOT EXISTS mv_monthly_stats AS
SELECT
    u.id AS user_id,
    u.name AS user_name,
    c.parent_id  AS parent_cat_id,
    c.name       AS parent_cat_name,
    c.color      AS parent_cat_color,
    date_trunc('month', br.trans_time) AS month,
    br.platform,
    COUNT(*) AS tx_count,
    SUM(br.amount) AS total_amount
FROM bill_records br
JOIN users u ON u.id = br.user_id
LEFT JOIN categories c ON c.id = br.category_id
WHERE br.direction = '支出'
GROUP BY u.id, u.name, c.parent_id, c.name, c.color, month, br.platform
WITH NO DATA;

-- 每月自动刷新
CREATE OR REPLACE FUNCTION refresh_mv_monthly_stats() RETURNS TRIGGER AS $$
BEGIN
    REFRESH MATERIALIZED VIEW CONCURRENTLY mv_monthly_stats;
    RETURN NULL;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trg_refresh_stats
    AFTER INSERT OR UPDATE OR DELETE ON bill_records
    FOR STATEMENT EXECUTE FUNCTION refresh_mv_monthly_stats();
```

## 四、自动分类策略

### 4.1 分类匹配流程

```
解析原始记录
    │
    ├─ 支付宝 ──→ 已有交易分类 ──→ 映射到用户分类体系
    │                  │
    │                  ├─ 精确匹配：支付宝分类名 == 用户分类名 → 直接分配
    │                  └─ 无匹配 → 降级到关键词匹配
    │
    ├─ 微信 ──→ 商户名 + 商品描述 → 关键词匹配
    │
    └─ 抖音 ──→ 交易对方（电商/生活服务）→ 关键词匹配（信息有限，依赖后续修正）
         │
         └─ 抖音电商 → 购物
         └─ 抖音生活服务 → 餐饮/娱乐/出行（取决于具体商户）
```

### 4.2 关键词匹配算法

```python
def classify_record(merchant: str, product: str, platform: str, categories: list) -> dict:
    """
    基于关键词的多层级分类匹配

    返回: { category_id, score, matched_keyword }
    """
    text = f"{merchant} {product}".lower()
    best_match = None
    best_score = 0

    for cat in categories:
        # 跳过二级分类的顶层匹配
        if cat.get("parent_id"):
            continue

        for kw in cat.get("keywords", []):
            score = _calculate_score(text, kw, merchant, product)
            if score > best_score:
                best_score = score
                best_match = {
                    "category_id": cat["id"],
                    "score": score,
                    "keyword": kw
                }

    # 阈值控制：低于阈值标记为"未分类"
    if best_score < 0.5:
        return {"category_id": None, "score": 0, "matched_keyword": None}

    return best_match


def _calculate_score(text: str, keyword: str, merchant: str, product: str) -> float:
    """
    综合评分：
    - 商户名完全匹配: 1.0
    - 商户名包含关键词: 0.8
    - 商品描述包含关键词: 0.6
    - 全文包含关键词: 0.4
    """
    if merchant == keyword:
        return 1.0
    if keyword in merchant:
        return 0.8
    if product and keyword in product:
        return 0.6
    if keyword in text:
        return 0.4
    return 0
```

## 五、账单解析器设计

### 5.1 解析器接口

```python
from abc import ABC, abstractmethod
from dataclasses import dataclass
from enum import Enum

class Platform(Enum):
    ALIPAY = "alipay"
    WECHAT = "wechat"
    DOUYIN = "douyin"

@dataclass
class RawRecord:
    platform: Platform
    trans_time: datetime
    amount: float
    direction: str  # "支出"|"收入"|"不计收支"
    merchant: str
    product: str
    platform_order_no: str
    merchant_order_no: str
    extra_fields: dict  # 各平台特有字段

class BaseParser(ABC):
    @abstractmethod
    def parse(self, file_path: str) -> tuple[list[RawRecord], dict]:
        """
        返回: (记录列表, 元数据)
        元数据: {
            "user_identifier": "手机号/昵称/证件号",
            "period_start": date,
            "period_end": date,
            "total_count": int
        }
        """
        ...
```

### 5.2 支付宝解析器

```python
class AlipayParser(BaseParser):
    SKIP_LINES = 22  # 前22行元数据
    ENCODING = "gbk"

    FIELD_MAP = {
        "交易时间": "trans_time",
        "交易分类": "original_category",
        "交易对方": "merchant",
        "对方账号": "extra_account",
        "商品说明": "product",
        "收/支": "direction",
        "金额": "amount",
        "收/付款方式": "payment_method",
        "交易状态": "status",
        "交易订单号": "platform_order_no",
        "商家订单号": "merchant_order_no",
        "备注": "extra_note",
    }

    def parse(self, file_path):
        records = []
        with open(file_path, encoding=self.ENCODING) as f:
            lines = f.readlines()

        # 提取元数据
        meta = self._extract_metadata(lines[:self.SKIP_LINES])

        # 解析 CSV 数据行
        header_line = lines[self.SKIP_LINES].strip().rstrip('\r\n')
        headers = [h.strip() for h in header_line.split(',')]

        for line in lines[self.SKIP_LINES + 1:]:
            line = line.strip().rstrip('\r\n')
            if not line:
                continue
            values = [v.strip() for v in line.split(',')]
            if len(values) != len(headers):
                continue
            record_dict = dict(zip(headers, values))
            records.append(self._to_raw_record(record_dict))

        return records, meta
```

### 5.3 微信解析器

```python
class WechatParser(BaseParser):
    SKIP_LINES = 17  # 前17行元数据

    def parse(self, file_path):
        wb = openpyxl.load_workbook(file_path, read_only=True)
        ws = wb.active

        # 提取元数据（从行1-17）
        meta = self._extract_metadata_from_excel(ws)

        # 从第18行开始解析
        records = []
        for row_idx, row in enumerate(ws.iter_rows(values_only=True)):
            if row_idx < 17:
                continue
            if len(row) < 10:
                continue
            record = {
                "trans_time": row[0],
                "transaction_type": row[1],
                "merchant": row[2] or "",
                "product": row[3] or "",
                "direction": row[4],
                "amount": float(row[5]) if row[5] else 0,
                "payment_method": row[6] or "",
                "status": row[7] or "",
                "platform_order_no": row[8] or "",
                "merchant_order_no": row[9] or "",
                "note": row[10] or "",
            }
            records.append(self._to_raw_record(record))

        return records, meta
```

### 5.4 抖音解析器

```python
class DouyinParser(BaseParser):
    def parse(self, file_path):
        reader = PdfReader(file_path)
        records = []

        for page in reader.pages:
            text = page.extract_text()
            lines = text.split('\n')

            # 第一页提取元数据
            if not records and lines:
                meta = self._extract_pdf_metadata(lines)

            for line in lines:
                record = self._parse_pdf_line(line)
                if record:
                    records.append(record)

        return records, meta

    def _parse_pdf_line(self, line: str) -> RawRecord | None:
        """
        PDF 行解析策略：
        - 跳过标题行、说明文字
        - 匹配包含时间戳的行（2026-XX-XX HH:MM:SS）
        - 使用正则提取固定列
        """
        # 示例正则（需要根据实际 PDF 格式微调）
        pattern = r'(\d{4}-\d{2}-\d{2}\s+\d{2}:\d{2}:\d{2})\s+(支出|收入|其他)\s+([\d.]+)\s+(.+?)\s+(\d+)\s+(.+?)\s+(\d+)'
        m = re.search(pattern, line)
        if not m:
            return None
        return RawRecord(
            platform=Platform.DOUYIN,
            trans_time=datetime.strptime(m.group(1), "%Y-%m-%d %H:%M:%S"),
            amount=float(m.group(3)),
            direction=m.group(2),
            merchant=m.group(6),  # 抖音电商/抖音生活服务
            product="",           # PDF 无商品字段
            platform_order_no=m.group(5),
            merchant_order_no=m.group(7),
            extra_fields={}
        )
```

### 5.5 用户识别与关联

```python
def identify_user(meta: dict, users: list) -> User | None:
    """
    根据账单元数据中的标识符匹配用户
    优先级: phone > name/nickname > document_id
    """
    identifier = meta.get("user_identifier", "")

    for user in users:
        if user.phone and identifier in user.phone:
            return user
        if user.name and user.name in identifier:
            return user
    return None
```

## 八、关键技术细节

### 8.1 文件大小限制

| 平台 | 典型大小 | 建议限制 |
|------|---------|---------|
| 支付宝 CSV | ~45KB | 无特殊限制 |
| 微信 Excel | ~320KB | 无特殊限制 |
| 抖音 PDF | ~310KB | 单文件 ≤ 10MB |

### 8.2 重复检测

```sql
-- 基于平台订单号去重
CREATE UNIQUE INDEX idx_records_unique_order
    ON bill_records(user_id, platform, platform_order_no)
    WHERE platform_order_no IS NOT NULL AND platform_order_no != '';
```

上传时检查 `platform_order_no` 是否已存在，避免重复导入。

### 8.3 数据一致性

- 每次解析前先标记旧文件为 `status='done'` 且 `parsed_at` 相同
- 使用事务：解析成功后一次性批量 INSERT bill_records
- 提供"重新解析"功能：删除旧记录后重新上传

### 8.4 安全

- Supabase RLS 确保用户只能访问自己的数据
- 文件上传限制 MIME 类型（csv/xlsx/pdf）
- 速率限制：每个用户每分钟最多上传 5 次
- 账单文件中的敏感信息（证件号、完整手机号）在解析后立即脱敏存储
