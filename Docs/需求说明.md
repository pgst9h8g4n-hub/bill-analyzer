# 需求说明文档

## 一、项目背景

用户在日常消费中使用支付宝、微信、抖音三个平台支付。各平台导出的账单文件格式不同、字段不同、分类维度不同，难以统一管理。本项目旨在将三个平台的账单统一解析、分类、统计，并通过 Web 端直观展示。

## 二、核心目标

**消费账单的分类统计与分析**，具体包括：

1. **统一解析**：自动识别并解析支付宝(CSV)、微信(Excel)、抖音(PDF)三种格式的账单文件
2. **自动分类**：基于关键词匹配，将每笔交易自动归类到用户自定义的消费分类
3. **手动修正**：用户可手动调整任意交易的分类
4. **统计分析**：按用户、分类、月份等多维度统计支出，饼图+柱状图可视化
5. **多用户支持**：家庭成员各自上传账单，系统按人区分统计
6. **Web 展示**：手机端友好，随时随地查看

## 四、数据格式说明

### 4.1 支付宝（GBK 编码 CSV）

- 前 22 行：元数据（姓名、手机号、时间范围等）
- 第 23 行：表头
- 后续行：交易记录
- 关键字段：交易时间、交易分类（16种自带分类）、交易对方、商品说明、收/支、金额

### 4.2 微信（Excel .xlsx）

- 前 17 行：元数据（昵称、时间范围等）
- 第 18 行：表头
- 后续行：交易记录
- 关键字段：交易时间、交易类型、交易对方、商品、收/支、金额(元)、支付方式

### 4.3 抖音（PDF）

- 3 页，每页多条记录
- 关键字段：交易时间、收/支、金额、收/付款方式、交易单号、交易对方（抖音电商/抖音生活服务）

## 六、非功能性需求

| 类别 | 要求 |
|------|------|
| 性能 | 单次解析不超过 30 秒；页面加载不超过 2 秒 |
| 安全 | 用户数据隔离（RLS）；敏感信息脱敏存储 |
| 兼容 | 支持 Chrome/Safari/Edge 最新两个版本；移动端 Safari/Chrome |
| 存储 | 原始账单文件保留 6 个月；交易记录永久保存 |
| 部署 | Supabase + Railway + Cloudflare Pages，月成本控制在免费额度内 |
