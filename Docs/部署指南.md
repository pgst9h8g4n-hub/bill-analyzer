## Railway 部署（3 步搞定）

### 第 1 步：创建项目
1. 打开 https://railway.app
2. 登录你的账号
3. 点 **New Project** → **Deploy from GitHub repo**
4. 授权 GitHub，选择 `消费账单记录` 仓库
5. 如果仓库不在 GitHub 上，选 **Empty Project**

### 第 2 步：添加 PostgreSQL 数据库
1. 在项目页面，点 **+ New**
2. 选 **Database** → **Postgres**
3. 等它创建完成（大约 1-2 分钟）
4. 创建好后，点这个 Postgres 服务的 **Settings** → **Variables**
5. 复制 **URI** 的值（格式类似 `postgresql://...`），后面要用

### 第 3 步：添加后端服务并配置变量
1. 回到项目主页，点 **+ New** → **Blank Service**
2. 给它起名 `backend`
3. 点这个服务的 **Settings** → **Variables**，添加以下变量：

| 变量名 | 值 | 来源 |
|--------|-----|------|
| `SUPABASE_URL` | `https://spzxyecryqenludsrrrn.supabase.co` | 已知 |
| `SUPABASE_SERVICE_ROLE_KEY` | `eyJhbGc...KVfQ.eqQCi...` | 你给我的 |
| `DATABASE_URL` | 从 Postgres 服务的 Settings → Variables 复制 URI | 上一步 |
| `SECRET_KEY` | `cc0c3c4c3cf81d535834dfe7bc8842be0487c461ae7615c3dd15ace101955a02` | 自动生成 |
| `ALLOWED_ORIGINS` | `https://bill-analyzer-332.pages.dev` | 前端网址 |

4. 点 **Deploy** 按钮

### 第 4 步：获取后端 URL
部署完成后，Railway 会给你一个 URL，类似：
```
https://backend-xxxx.up.railway.app
```
把这个 URL 告诉我，我更新前端的 API 配置。

## 常见问题

**Q: 部署失败怎么办？**
A: 点服务 → Logs 查看错误日志，通常是环境变量没配对。

**Q: 数据库连不上？**
A: 确认 DATABASE_URL 是从 Postgres 服务的 Settings → Variables 里复制的。

**Q: 端口不对？**
A: Railway 会自动设置 PORT 环境变量，我们的 Procfile 已经配置好了。
